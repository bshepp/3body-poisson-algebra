# SageMath oracle — planar 3-body Poisson algebra

Independent (third-leg) CAS verification of the central result of this
codebase:

> The Lie algebra generated by pairwise interaction Hamiltonians under the
> Poisson bracket has cumulative dimension sequence **[3, 6, 17, 116]** for
> N=3 bodies in d=2, for both 1/r (Newtonian) and 1/r² (Calogero-Moser)
> potentials.

This directory mirrors the structure of [`../mathematica/`](../mathematica/)
in SageMath. Three independent implementations now produce the same
numbers:

| Leg | CAS | Rank algorithm | Phase in [`validation_summary.md`](../bench_flint/validation_summary.md) |
|-----|-----|---------------|-----|
| 1 | Python (SymPy ≥ 1.13.3) | `DomainMatrix.rank()` over QQ | E |
| 2 | Wolfram Mathematica 14.3 | `MatrixRank` over `Rationals` on `SparseArray` | F |
| 3 | SageMath | `Matrix(QQ, ..., sparse=True).rank()` (FLINT-backed) | G.1 |

The Lane C streaming mod-p L=4 consumer is Phase G.2.

## Files

| File | Purpose |
|------|---------|
| [`poisson_n3_d2_engine.sage`](poisson_n3_d2_engine.sage) | Shared engine — Poisson bracket with chain rule on `u_ij = 1/r_ij`, Lie closure with frontier-based dedupe, exact rank via sparse `Matrix(QQ, ...).rank()` |
| [`poisson_n3_d2.sage`](poisson_n3_d2.sage) | Sanity runner — L=3 for both 1/r and 1/r² and checks `[3, 6, 17, 116]` |

## Convention

Identical to the Mathematica engine; see
[`../mathematica/README.md`](../mathematica/README.md) for the chain-rule
and filtration details. The Sage engine uses the same variable names
(`x_i`, `y_i`, `px_i`, `py_i`, `u_ij`), the same Hamiltonian convention,
and the same level-by-level dedupe.

Where Mathematica uses `Together` to combine fractions, the Sage engine
operates on the `FractionField(PolynomialRing(QQ, ...))` and clears
denominators (which are pure `u_ij` monomials for our brackets) before
taking ranks. This is mathematically equivalent.

## Running

You need a SageMath installation. On Linux/macOS:

```bash
sage sage/poisson_n3_d2.sage          # L=3 sanity, both potentials
sage sage/poisson_n3_d2_harmonic.sage # L=4 harmonic closure
```

On Windows the canonical paths are WSL or a conda env:

```powershell
# Inside a conda env where sagemath was installed
sage sage\poisson_n3_d2.sage
sage sage\poisson_n3_d2_harmonic.sage
```

JSON outputs:

- `sage/results/n3_d2_dimseq.json` — schema matches
  `mathematica/results/n3_d2_dimseq.json` with `wolfram_version` replaced
  by `sage_version` + `python_version`.
- `sage/results/n3_d2_harmonic.json` — schema matches
  `mathematica/results/n3_d2_harmonic.json` with the same key swap.

Cross-CAS diff:

```bash
python -c "
import json
m = json.load(open('mathematica/results/n3_d2_dimseq.json'))
s = json.load(open('sage/results/n3_d2_dimseq.json'))
for pot in m['results']:
    assert m['results'][pot]['cumulative_rank'] == s['results'][pot]['cumulative_rank'], pot
    assert m['results'][pot]['raw_gens_per_level'] == s['results'][pot]['raw_gens_per_level'], pot
print('MATCH: SymPy = Mathematica = SageMath')
"
```

## Notes

- This is a pure CAS computation — no random sampling, no SVD, no
  numerical tolerances. Correctness rests on Sage's `Matrix.rank()` over
  QQ (FLINT under the hood).
- For N=3 d=2 a single kernel suffices. For larger N you'd want
  `@parallel` over the bracket evaluation loop.
- Sage runs are typically a few minutes for L=3 and minutes-to-tens-of-
  minutes for the L=4 harmonic closure.

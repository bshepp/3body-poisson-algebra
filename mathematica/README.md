# Mathematica oracle вҖ” planar 3-body Poisson algebra

Independent CAS verification of the central result of this codebase:

> The Lie algebra generated by pairwise interaction Hamiltonians under the
> Poisson bracket has cumulative dimension sequence **[3, 6, 17, 116]** for
> N=3 bodies in d=2, for both 1/r (Newtonian) and 1/rВІ (Calogero-Moser)
> potentials.

This directory contains a self-contained Mathematica reproduction that uses:
- a different CAS (Wolfram Language vs SymPy);
- a different rank algorithm (`MatrixRank` over `Rationals` on a `SparseArray`
  vs SymPy's `DomainMatrix`);
- but the **same** Poisson-bracket convention and the **same** Lie-closure
  filtration as `nbody/symbolic_rank_nbody.py`.

Matching dimension sequences across two independent implementations is a
strong correctness check for the headline numbers. The match is recorded in
[../bench_flint/validation_summary.md](../bench_flint/validation_summary.md)
once the runs complete.

## Convention details

**Phase-space variables** for body `i`:
`x_i, y_i, px_i, py_i` (planar, d=2).

**Auxiliary variables** for inverse pair distance:
`u_ij = 1 / sqrt((x_i - x_j)ВІ + (y_i - y_j)ВІ)` for `i < j`.

We treat `u_ij` as opaque symbols and apply the chain rule manually:

```
вҲӮu_ij/вҲӮx_k = -(x_i - x_j) u_ijВі (Оҙ_ki - Оҙ_kj)
вҲӮu_ij/вҲӮy_k = -(y_i - y_j) u_ijВі (Оҙ_ki - Оҙ_kj)
вҲӮu_ij/вҲӮp_k = 0
```

**Pairwise Hamiltonian** (unit masses):

```
H_ij = (px_iВІ + py_iВІ)/2 + (px_jВІ + py_jВІ)/2 + V(u_ij)
```

with `V(u) = -u` for 1/r, `V(u) = -uВІ` for 1/rВІ.

**Poisson bracket**:

```
{f, g} = ОЈ_k (вҲӮf/вҲӮq_k В· вҲӮg/вҲӮp_k вҲ’ вҲӮf/вҲӮp_k В· вҲӮg/вҲӮq_k)
```

over all 6 (q, p) pairs, with `Together` applied at the end.

**Filtration** (matches Python engine `build_generators`):

- L0: `{H_12, H_13, H_23}` вҖ” 3 generators.
- L1: brackets of every level-0 pair: `{{H_12, H_13}, {H_12, H_23}, {H_13, H_23}}` вҖ” 3 new candidates.
- L_k (k вүҘ 2): for each frontier element (level k-1) and each prior
  generator (levels 0..k-1), if the pair has not been bracketed before,
  compute `PB[i, j]` and add to the pool. Pairs are dedupe'd via a
  frozenset-equivalent (sorted `{i, j}` keyed in an `Association`).

**Exact rank**: each generator is `Together`-simplified, then expanded;
`CoefficientRules[expr, allVars]` gives `{monomial -> coeff}`. We assemble
a `SparseArray` of all (generator, monomial) coefficients and call
`MatrixRank`, which works over the rational field by default.

## Running

You need either the Wolfram Engine (free) or Mathematica with
`wolframscript` on `PATH`.

### Sanity check (L=3, both potentials, ~minutes)

```powershell
cd f:\science-projects\3body
wolframscript -file mathematica\poisson_n3_d2.wl
```

Expected stdout (abridged):

```
[1/r] generating algebra up to level 3
  L0: 3 gens, cumulative rank = 3
  L1: +3 raw, cumulative rank = 6
  L2: frontier=3 existing=6 candidates=12
    cumulative rank = 17
  L3: frontier=12 existing=18 candidates=126
    cumulative rank = 116
[1/r] done

SUMMARY
1/r:    cumulative_rank = {3, 6, 17, 116}   expected {3, 6, 17, 116}   MATCH
1/r^2:  cumulative_rank = {3, 6, 17, 116}   expected {3, 6, 17, 116}   MATCH
```

JSON output: `mathematica/results/n3_d2_dimseq.json`.

### L=4 backup (1/r only, hours)

```powershell
wolframscript -file mathematica\poisson_n3_d2_l4_backup.wl
```

Output: `mathematica/results/n3_d2_l4_1r.json`. The L=4 entry of
`cumulative_rank` answers the open question of whether the algebra closes
at 116 (L=3 result) or extends.

## Notes

- `Together` is the Mathematica analogue of the `together` patch we landed
  in Phase E for the SymPy engine вҖ” same algebraic effect, same
  filling-out of common denominators.
- For larger N or d, you'd want `Parallelize` and `ParallelTable` over the
  `workItems` loop, but for N=3 d=2 a single kernel suffices.
- This is a pure CAS computation вҖ” no random sampling, no SVD, no
  numerical tolerances. The only correctness assumption is `MatrixRank`
  over `Rationals`, which Wolfram has battle-tested for decades.
